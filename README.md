# 🎡 Колесо рандома

Колесо случайного выбора с режимом **на выбывание**. React + Vite.

- Списки лотов (добавить по одному / вставить списком / редактировать / удалить)
- Режим «на выбывание»: победитель убирается из колеса до последнего
- История победителей с датой/временем
- Всё хранится в **localStorage** браузера — бэкенд не нужен

## Локально

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # сборка в dist/
npm run preview  # посмотреть готовую сборку
```

## Деплой (бесплатно)

Данные лежат в браузере пользователя, сервер не нужен — подойдёт любой статический хостинг.

### Vercel (рекомендую)
1. Залей папку в репозиторий на GitHub.
2. На vercel.com → **Add New Project** → импортируй репозиторий.
3. Framework определится как Vite автоматически. Деплой. Готово.
   (Конфиг уже есть в `vercel.json`.)

### Netlify
1. Репозиторий на GitHub.
2. netlify.com → **Add new site → Import** → выбери репозиторий.
3. Build command `npm run build`, publish `dist` (уже в `netlify.toml`). Deploy.

### Cloudflare Pages
Похоже на Netlify: build `npm run build`, output `dist`.

### GitHub Pages
В Pages нужен базовый путь репозитория:
```bash
# Windows PowerShell
$env:VITE_BASE="/<имя-репозитория>/"; npm run build
```
Затем опубликуй содержимое `dist/` в ветку `gh-pages` (или через GitHub Action).

> Примечание: history и списки хранятся в localStorage конкретного браузера/устройства.
> На другом устройстве данных не будет — это локальное хранилище, не общая база.

## Гифки в центре (Vercel Blob)

Гифки хранятся в облаке **Vercel Blob**, поэтому они доступны на всех устройствах
и без лимита 5 МБ. Нужно один раз подключить хранилище:

1. На Vercel открой проект → вкладка **Storage** → **Create Database** → **Blob** → создай стор.
2. При создании выбери привязку к этому проекту — Vercel сам добавит переменную
   окружения **`BLOB_READ_WRITE_TOKEN`** в проект.
3. Сделай **Redeploy** (или просто запушь любой коммит). Готово.

После этого кнопка **🖼 Гифка** открывает плашку: загрузить новую гифку, выбрать
активную (она крутится в центре колеса) или удалить.

Папки `/api/*.js` — это serverless-функции Vercel (`upload`, `list`, `delete`),
они работают только на задеплоенном сайте. Локально для них нужен `vercel dev`
(`npm i -g vercel`, затем `vercel dev`); обычный `npm run dev` поднимает только фронт.
